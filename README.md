
# Plato Coffee-Shop-Sales
To analyze sales performance, identify customer purchasing patterns, track product demand, and generate insights that help optimize pricing strategies, inventory management, and overall business decision-making.

# Problem Satement
The Plato Coffee shop is a small coffe shop which generated large amounts of sales data from daily transactions for the past 3 years, the coffe shop always finds itself in failures in decision-making:
- Reactive Inventory Management
- Inefficient Staffing
- Marketing
  
# Aim
The primary aim of this project is to design and implement a data-driven decision-making framework for Plato Coffee Shop that transforms three years of raw        transactional data into actionable business intelligence. By analyzing sales data, this project seeks to address the core operational failures identified in       the problem statement through the following specific:


# Objective
- Use Miro For Brainstorming
- Use Canva For project planning
- Use data bricks for data processing


# Summary of Results
- Optimize Inventory Management: To transition from reactive stock management to predictive inventory control.
- Enhance Staffing Efficiency: To align labor allocation with actual customer traffic patterns.
- Enable Targeted Marketing: To leverage customer purchase history for personalized engagement and retention.
